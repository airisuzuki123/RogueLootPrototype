# Rogue Loot Prototype

一个 Godot 4 + GDScript 的小型肉鸽刷宝原型工程。

## 当前目标

- 先完成可跑、可验收的单局局内体验。
- 保持代码结构简单，方便学习、拆解和重构。
- 后续按阶段加入单局流程、关卡事件、精英、Boss、局外成长、内容平衡、美术资源和音效资源。

## 当前开发阶段

- 阶段 2：装备与构筑原型已暂时完成，后续只做验收返修。
- 阶段 3：单局流程结构与关卡节奏已重新定位为弹幕式怪物与走位躲避原型；当前保留阶段流程骨架，下一步改造弹幕型敌人与阶段主题。
- 既定 8 阶段计划记录在 `docs/stage-plan.md`，后续开发按该阶段顺序推进，短期内不再调整阶段结构。

## 目录结构

- `scenes/`：Godot 场景文件。
- `scripts/core/`：全局管理与流程控制。
- `scripts/entities/`：玩家、敌人等实体逻辑。
- `scripts/items/`：掉落物、装备、道具逻辑。
- `docs/`：设计说明和学习记录。

## 本地运行

1. 安装 Godot 4.x。
2. 用 Godot 打开 `RogueLootPrototype/project.godot`。
3. 运行主场景 `scenes/main.tscn`。

## 初始操作

- `WASD`：移动玩家。
- 玩家会自动向最近的敌人发射投射物。
- 敌人会持续生成并追踪玩家。
- 敌人包含普通、小型快速、坦克和远程类型。
- 敌人接近玩家后会造成接触伤害。
- 远程、游走扫射和阵列发射敌人会保持距离并发射不同弹幕。
- 玩家和敌人受击会产生短距离击退。
- 敌人受伤后会显示血条，便于判断击杀进度。
- 击杀敌人后掉落经验球，并有较低概率额外掉落金币或装备。
- 拾取经验球获得经验并触发升级选择。
- 拾取装备会放入背包，按 `B` 打开背包时暂停战斗并比较同槽位装备，按 `Esc` 或 `B` 可关闭。
- 背包容量为 36 格，满容量后装备掉落不会被拾取；清出容量后可重新拾取地上的装备。
- 装备分解金币由品质和评分共同决定。
- 升级时选择一个强化，例如伤害、攻速、移速、生命、多重射击或当前武器形态专精。

## 当前可玩内容

- HUD 显示生命、等级、经验、金币和击杀数。
- HUD 显示当前穿戴装备和最近拾取信息。
- 装备有普通、魔法、稀有 3 档品质，当前支持武器、头盔、护甲、鞋子、项链、戒指 6 个槽位。
- 武器形态包含聚能、散射、穿透和爆裂法杖。
- 装备词缀目前支持伤害、最大生命、攻速、移速、暴击率、吸血、金币获取、额外投射物、穿透和爆裂范围，并按槽位限制可出现词缀。
- 升级选项会根据当前装备的武器形态加入对应专精，例如聚能增伤、散射加投射物、穿透加穿透、爆裂加范围。
- 装备拾取后进入背包，背包内按品质和等级排序。
- 经验球使用蓝绿色圆形并标“经”，金币使用黄色圆币并标“金”，装备使用品质颜色大菱形并标槽位字，便于区分。
- 打开背包后左侧显示当前穿戴装备和构筑总览，右侧显示容量、品质分解勾选、分解选中按钮和带品质边框的背包装备图标网格。
- 点击图标会打开装备详情弹窗，当前装备和选中装备左右并列显示，可查看推荐结论、评分变化和核心属性变化，并选择装备、卸下或分解获得金币；详情内容可滚动，点击弹窗外区域可关闭。
- 单局会按时间推进到不同阶段，HUD 显示当前阶段、目标、剩余时间、下一阶段预告、阶段击杀进度和阶段奖励；当前阶段流程骨架后续会承载弹幕式怪物与走位躲避主题。
- 达成当前阶段击杀目标后会立刻获得少量金币、经验或回复，并显示阶段目标完成提示。
- 阶段临近结束时会弹出倒计时提示，提前告知下一阶段或本局完成。
- 撑过终局试炼后会完成本局并显示完成结算；死亡仍显示失败结算。
- 远程敌人会按当前阶段从弹幕池中发射直线、集束、扇形、环形、交叉、螺旋、双层环、花形或弹幕墙；游走扫射者偏扇形/交叉，阵列发射者偏环形/花形。
- 掉落装备概率和金币数量会随玩家等级小幅提升。
- 玩家死亡或完成本局后会停止刷怪并显示结算。
- 项目目前仍是原型阶段，关卡事件、Boss、局外成长、更多内容平衡、美术资源和音效资源尚未实现。

## 当前体验关注点

- 移动边界是否自然。
- 敌人生成频率和弹幕密度是否过快或过慢。
- 不同敌人类型和弹幕模式是否能形成不同走位压力。
- 受击无敌和击退是否让近身压力更清晰。
- 弹幕密集时玩家、敌方弹体、玩家弹体和掉落物是否仍然清晰可读。
- 经验球拾取节奏和装备掉落是否足够清晰。
- 装备评分、分解金币、背包容量和背包暂停是否让拾取决策足够清楚。

## 下一步开发建议
