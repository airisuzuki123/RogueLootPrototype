# Rogue Loot Prototype

一个 Godot 4 + GDScript 的小型肉鸽刷宝原型工程。

## 当前目标

- 先完成可跑的 2D 生存战斗循环。
- 保持代码结构简单，方便学习、拆解和重构。
- 后续逐步加入装备词缀、掉落稀有度、局外成长和关卡事件。

## 目录结构

- `scenes/`：Godot 场景文件。
- `scripts/core/`：全局管理与流程控制。
- `scripts/entities/`：玩家、敌人等实体逻辑。
- `scripts/items/`：掉落物、装备、道具逻辑。
- `docs/`：设计说明和学习记录。

## 本地运行

1. 安装 Godot 4.x。
2. 用 Godot 打开 `RogueLootPrototype/project.godot`。
3. 运行主场景 `scenes/main.tscn`。

## 初始操作

- `WASD`：移动玩家。
- 玩家会自动向最近的敌人发射投射物。
- 敌人会持续生成并追踪玩家。
- 敌人包含普通、小型快速、坦克和远程类型。
- 敌人接近玩家后会造成接触伤害。
- 远程敌人会保持距离并发射投射物。
- 玩家和敌人受击会产生短距离击退。
- 敌人受伤后会显示血条，便于判断击杀进度。
- 击杀敌人后获得经验，并有概率掉落金币或装备。
- 拾取装备会放入背包，按 `B` 打开背包时暂停战斗并比较同槽位装备，按 `Esc` 或 `B` 可关闭。
- 装备分解金币由品质和评分共同决定。
- 升级时选择一个强化，例如伤害、攻速、移速、生命或多重射击。

## 当前可玩内容

- HUD 显示生命、等级、经验、金币和击杀数。
- HUD 显示当前武器、护甲、饰品和最近拾取信息。
- 装备有普通、魔法、稀有 3 档品质，当前支持武器、护甲、饰品 3 个槽位。
- 武器形态包含聚能、散射、穿透和爆裂法杖。
- 装备词缀目前支持伤害、最大生命、攻速、移速、暴击率、吸血、金币获取、额外投射物、穿透和爆裂范围，并按槽位限制可出现词缀。
- 装备拾取后进入背包，背包内按品质和等级排序。
- 打开背包后可查看推荐结论、当前同槽位装备/选中装备并排对比、评分变化和核心属性变化，并选择装备或分解获得金币。
- 掉落装备概率和金币数量会随玩家等级小幅提升。
- 玩家死亡后停止刷怪并显示本局结算。
- 项目目前仍是原型阶段，更多装备词缀、局外成长和 Boss 尚未实现。

## 当前体验关注点

- 移动边界是否自然。
- 敌人生成频率是否过快或过慢。
- 不同敌人类型是否能形成不同走位压力。
- 受击无敌和击退是否让近身压力更清晰。
- 升级节奏和装备掉落是否足够频繁。
- 装备评分、分解金币和背包暂停是否让拾取决策足够清楚。
