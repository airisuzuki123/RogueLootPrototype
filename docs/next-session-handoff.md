## 重要设计约束

- 不要在 HUD、商店或升级按钮里显示路线名、标签名、续牌规则、权重规则或内部构筑概念。
- 不要给某一条路线写特殊保底逻辑；出货和收束优先用全局规则处理。
- 不要把复杂职业专精、天赋树、职业等级、局外资源或永久成长提前塞进阶段 5.5。
- 职业只做“开局导向器”：通过初始属性、少量乘区放大或轻量出货偏向，让玩家更早进入某种玩法倾向，但不能锁死 BD。
- 技能描述继续保持简洁，但必须写清具体数值、代价、条件和上限；不要提示重复拿取规则，玩家通过实际变化感受。
- 继续以 1600x900 作为主要体验基准，保留 F11 / Alt+Enter 全屏支持。

## 阶段 5.5 主要工作

1. 职业数据结构
   - 新增职业目录或数据模块，集中管理职业 id、名称、描述、初始属性偏向和乘区效果。
   - 避免把职业逻辑散落在 Player、GameManager 和 UI 多处。

2. 开局职业选择入口
   - 在开始本局前显示职业选择。
   - 第一版只需要选择后进入当前 10 关短循环。
   - UI 只展示玩家能理解的实际效果，不展示内部权重或路线标签。

3. 第一批职业
   - 巨躯炮台：偏大体积、慢速、体积转伤害。
   - 沉稳射手：偏静立、暴击和稳定输出。
   - 贴身刃卫：偏护盾、近身刀环和脉冲场。
   - 游走电弧：偏高速、小体积、连锁/回旋/追踪弹道。
   - 重弹爆破：偏爆裂、重弹和低攻速高单发。

4. 职业效果接入
   - 装备仍提供基础属性。
   - 技能继续负责百分比放大和触发器。
   - 职业只在开局给少量偏向或对特定已存在机制做乘区放大。

5. 验收与自测
   - 每个职业前 2 关应能感到开局差异。
   - 职业不能让某一构筑无脑最优。
   - 职业不能导致商店和升级只出同一类技能。
   - 阶段 5 的 10 关短循环、商店、升级、背包和结算不能被破坏。
