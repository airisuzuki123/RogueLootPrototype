# 阶段开发计划

本文档是项目后续开发的标准阶段计划。当前版本固定为 8 个阶段，短期内不再调整阶段数量、顺序和目标；除非用户明确要求重新规划，否则中途不主动修改阶段结构。

## 执行规则

- 阶段内可以做 bug 修复、体验返修和必要的小范围补齐。
- 明显超出当前阶段的新需求，先归档到对应未来阶段，不直接插入当前开发。
- 本版 8 阶段计划纳入项目规则，后续开发以本文档为准。
- 只有用户明确批准重新规划阶段时，才修改本文档的阶段结构。
- 提交前必须验证提交内容；涉及 Godot gameplay、scene、script、preload path 或 Autoload 的改动，必须通过 Godot headless validation 后再提交。

## 阶段 1：基础可玩循环

状态：已完成。

目标：

- 建立 2D 生存战斗主循环。
- 实现玩家移动、自动攻击、敌人生成、掉落、经验、升级和死亡结算。
- 加入基础战斗反馈，让原型可试玩、可判断节奏。

已完成内容：

- 玩家移动和屏幕边界限制。
- 自动攻击、投射物、敌人追踪和接触伤害。
- 普通、快速、坦克、远程敌人。
- 金币、经验、升级三选一和死亡结算。
- 伤害数字、命中特效、受击闪烁、击退和敌人血条。

## 阶段 2：装备与构筑原型

状态：暂时完成，进入验收返修口径。

目标：

- 建立装备掉落、背包、穿戴、分解和同槽位比较流程。
- 扩展装备槽位、品质、词缀和武器形态，让玩家能形成初步构筑。
- 让装备选择不频繁打断战斗，背包决策在玩家主动打开时完成。

已完成内容：

- 装备拾取进入背包，不再立即暂停战斗。
- 敌人死亡掉落经验球，拾取经验球后获得经验并触发升级选择。
- 经验球、金币和装备通过形状、颜色和文字标识区分。
- `B` 打开背包并暂停战斗，`B` 或 `Esc` 关闭背包。
- 背包装备按品质、等级、评分排序。
- 装备槽位固定为武器、头盔、护甲、鞋子、项链、戒指。
- 背包采用左侧穿戴区、右侧图标网格、点击弹窗详情的布局。
- 背包左侧显示构筑总览，用于汇总总评分、武器形态和装备词缀总加成。
- 装备详情弹窗左右并列显示当前同槽位装备和选中装备，并显示推荐结论、评分变化和核心属性变化。
- 装备详情内容可滚动，点击弹窗外区域可关闭。
- 背包容量为 36 格，满容量后装备掉落无法继续拾取；清出容量后可重新拾取保留在场上的装备。
- 背包内装备可穿戴或分解，替换下来的旧装备回到背包。
- 当前穿戴装备可卸下回到背包，背包满时无法卸下。
- 背包支持勾选指定品质后分解选中品质。
- 装备品质包含普通、魔法、稀有。
- 武器形态包含聚能、散射、穿透、爆裂。
- 词缀包含伤害、最大生命、攻击速度、移动速度、暴击率、吸血、金币获取、额外投射物、穿透和爆裂范围。
- 升级池会按当前武器形态加入对应专精选项。
- 掉落概率、金币数量、装备评分和分解金币已有第一版调校。
- 装备掉率已下调，经验球承担主要成长掉落节奏。

验收返修范围：

- 拾取装备不会造成频繁停顿。
- 经验通过拾取经验球获得，经验球和装备外观清晰区分。
- 打开背包时战斗暂停，关闭后战斗继续。
- 背包装备排序符合品质 > 等级 > 评分。
- 六个装备槽位的穿戴、替换、分解和详情弹窗交互可用。
- 背包容量显示、满包拒绝拾取、满包后重新拾取、装备卸下和按品质分解选中可用。
- 装备词缀能正确影响玩家输出、生存、移动、吸血和金币收益。
- 武器形态和形态专精能形成可感知的差异。
- 构筑总览能帮助验收当前装备加成和构筑方向。
- 当前数值足够支撑一轮短时间试玩，不要求最终平衡。

## 阶段 3：弹幕式单局流程与关卡节奏

状态：重新规划中。原时间阶段、目标奖励和结算骨架保留，后续开发方向调整为弹幕躲避型关卡体验。

新定位：

- 单局核心压力从“近身刷怪堆数量”转向“华丽、可读、可躲的弹幕式怪物攻击”。
- 玩家主要体验应是封闭场地内的走位、穿缝、预判弹道和维持输出，而不是单纯绕圈清怪。
- 视觉语言参考弹幕射击和几何竞技场：高亮几何弹体、环形/螺旋/扫射/定向弹幕、清晰危险区域和弹幕层次。
- 装备和升级继续提供输出、生存和机动收益，但阶段 3 的首要验收对象是局内弹幕压力曲线。

目标：

- 建立弹幕式敌人和弹幕发射模式的第一版原型。
- 重做阶段压力曲线，让每个阶段引入一种明确的弹幕阅读与躲避主题。
- 保留当前单局阶段、目标、奖励、倒计时和完成结算骨架，用它承载新的弹幕关卡节奏。
- 为阶段 4 的精英、事件和 Boss 提供可复用的弹幕模式基础。

范围：

- 基础弹幕系统：敌方投射物形状、速度、寿命、碰撞、颜色和可读性。
- 弹幕模式：环形扩散、螺旋发射、定向连射、扇形扫射、横向/纵向弹幕墙。
- 弹幕型敌人：固定发射器、移动发射器、追踪压迫者、阵列发射者等原型。
- 阶段主题：每个阶段主打一类弹幕读法，并逐步叠加密度、速度和组合。
- 封闭竞技场体验：玩家移动空间、边界压力、弹幕密度和拾取物干扰的基础调校。
- HUD 和反馈：阶段主题、倒计时、目标、完成提示和结算继续可读。

已完成基础：

- 建立第一版单局时间阶段：初始清场、追击加压、混编压迫、密集来袭、终局试炼。
- HUD 显示单局时间、当前阶段、阶段剩余时间、下一阶段预告和阶段目标。
- 刷怪间隔、单次刷怪数量、敌人解锁等级和敌人权重会随当前阶段变化。
- 背包、升级选择等暂停状态下，单局计时和阶段推进暂停。
- 每个阶段有独立击杀目标，HUD 显示阶段击杀进度和阶段奖励。
- 达成阶段击杀目标后会立即获得少量局内奖励，并显示目标完成提示。
- 进入新阶段时会显示阶段转换提示。
- 阶段剩余 10 秒时会显示倒计时提示。
- 终局试炼有固定持续时间，撑过后会完成本局并显示完成结算。

待完成弹幕化改造：

- 将现有远程敌人扩展为多种弹幕发射模式，而不是只发射单发直线弹。
- 新增至少 4 类基础弹幕模式：环形、螺旋、扇形、弹幕墙。
- 调整阶段配置，让不同阶段按主题解锁和叠加弹幕模式。
- 降低纯近身追逐怪的压力占比，提高弹幕发射敌人与弹幕密度。
- 调整玩家碰撞、受击反馈和弹体视觉，保证弹幕密集时仍能判断安全缝隙。
- 验收拾取物、玩家弹体、敌方弹体同时出现时的画面可读性。

阶段 3 验收标准：

- 玩家死亡或受压主要来自弹幕躲避失败，而不是被普通敌人数量淹没。
- 每个阶段都有清晰不同的弹幕主题和走位要求。
- 弹幕足够华丽但仍可读，玩家能判断弹体方向、速度和安全缝隙。
- 背包、升级选择和本局结算仍能正确暂停或停止弹幕、敌人和玩家投射物。
- 当前数值只要求支撑短时间原型验收，不要求最终平衡。

不包含：

- 局外成长。
- 永久强化。
- 长期资源消耗。
- 精英、事件和 Boss。
- 正式美术资源和音效资源。
- 大规模数值平衡。

## 阶段 4：关卡事件、精英和 Boss

状态：未开始。

目标：

- 在阶段 3 的单局流程结构上加入更明确的战斗节点和奖励节点。
- 让装备构筑在更具体的战斗场景中接受检验。
- 提升单局过程中的变化感和目标感。

范围：

- 精英怪。
- Boss。
- 宝箱。
- 商店。
- 随机事件。
- 阶段性遭遇。
- 特殊奖励。

## 阶段 5：局外成长与资源消耗

状态：未开始。

目标：

- 在单局体验和关卡节点稳定后，再引入长期成长。
- 让一局内获得的部分资源能转化为局外成长。
- 明确局外成长与局内掉落、升级、装备之间的边界。

范围：

- 明确局外资源。
- 死亡结算后保留部分资源。
- 局外成长入口。
- 第一批永久强化。
- 强化消耗。
- 基础存档：资源和强化等级。

## 阶段 6：内容池扩充与数值平衡

状态：未开始。

目标：

- 扩展可重复游玩的内容量。
- 调整局内掉落、局内成长、装备收益和局外成长之间的数值关系。
- 解决内容不足和节奏不稳定的问题。

范围：

- 更多装备变体。
- 更多词缀。
- 更多升级选项。
- 更多敌人类型。
- 掉落率调整。
- 金币、经验、装备评分、分解收益平衡。
- 局外成长数值平衡。

## 阶段 7：美术资源与音效资源制作

状态：未开始。

目标：

- 在核心玩法、局内流程、局外成长和数值方向稳定后，统一制作与替换正式感更强的表现资源。
- 提升识别度、反馈感和整体完成度，但不提前用资源制作打断玩法验证。

范围：

- 玩家、敌人、投射物和掉落物美术资源。
- 装备、品质、技能和 UI 图标。
- 场景背景、地面、特效和基础动画。
- 攻击、受击、拾取、升级、分解、弹窗和结算音效。
- 背景音乐或基础氛围音。
- 资源导入规范和命名整理。

## 阶段 8：体验打磨、存档完善与发布准备

目标：

- 把原型整理成更完整、稳定、可持续试玩的版本。
- 补齐发布前必要的体验和稳定性工作。

范围：

- UI/UX 打磨。
- 设置菜单。
- 重开流程。
- 存档稳定性。
- 音效与视觉反馈完善。
